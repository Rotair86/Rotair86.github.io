DomainName Property

Used to return the Domain name.

Syntax
VB 
'Declaration
Public Shared Shadows ReadOnly Property DomainName
As String

Return Value

Type: System.String
The domain name.
Remarks

The domain name is the key to AppObj_MetadataHeader and can also be a subdomain.

The constructor requires this field as it must access the table with the meta data in it.

A domain needs the domain name when it is defining that domain as a subdomain when calling DefineSubdomains

It can be used to return the collection of objects when it represents a subdomain or to inactivate / reactivate to reload a subdomain. For these uses it is used in the same way, passed to a method to be used as a key to return the matching subdomain (collection of objects).

Used when joining a domain to another for the purpose of looking up fields.

In the SubdomainLoadSQL method the domain name (which would really be the subdomain name in this case) is used to determine what SQL to return.

Some controls require this information to function properly.

Example

Here the NewSetup method is overridden to create a new guid for the record id field and prepare two subdomains, 'ItemSupplemental' and 'ItemUseClass'.

VB 
Protected Overrides Function NewSetup() As imsReturnData

    Dim ao As imsAppObject
    Dim r2d2 As New imsReturnData()


    'Call the base to set the standard defaults.
    r2d2.Merge(MyBase.NewSetup)
    'Sets the field IMA_RecordID to a new guid.  This is already done in the base.
    '    It is only replicated here for example purposes.
    r2d2.Merge(SetValue(True, "IMA_RecordID", Guid.NewGuid))
    'Must be done for custom subdomains.
    ao = Me.ItemSupplemental_Collection.NewObject()
    ao = Me.ItemUseClass_Collection.NewObject()

    Return r2d2

End Function
