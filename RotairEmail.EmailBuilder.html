rotEmailBuilder Class

Sends out table based email. The functions are shared to eliminate the need for instantiating an object and also to self-document.

This module has many ways to test.

  1. There is a constant which can be set to TRUE. This is used by Skip House on his laptop and requires a re-compile to put in effect. This will cause messages to be displayed in a message box.
  2. There is a TestFlag property which can be set to TRUE. This can be done in code to switch back and forth between test mode and production. This will cause messages to be displayed in a message box.
  3. A TestRecipient property allows the recipient list to be overridden with a single test recipient.
  4. If the function IsProduction returns FALSE then e-mails are automatically routed to the logged in user and the recipient list is ignored.
  5. If the logged in user is equal to the constant mconLeadDeveloper then e-mails are automatically routed to the logged in user and the recipient list is ignored.

To use this class it requires certain tables to be populated. See Usage for details.

[Stored in a constant is the value localSMTP.corp.rotair.com which is the FQDN (Fully Qualified Domain Name) the host resolves in the internal DNS.]

To use this class see Usage for instructions.
