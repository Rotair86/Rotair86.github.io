Assemblies containing forms inheriting from vendor supplied forms whose base is imsForm

These are the assemblies that are derived from a vendor supplied form which is in turn derived from imsForm. The framework finds the customizations by the _CUSTOM suffix. These classes require the custom bit to be flipped in the table. The namespaces are named identically to the assemblies and can be considered as one since the vendor has not introduced additional namespaces.

Following is a list of the assemblies in alphabetical order.
Customized assemblies of vendor supplied forms
Name Description
IntuitiveUI_Customer_CUSTOM For a customer, the modified customer card:
  1. Forces screening to be current before any modifications can be saved.
  2. Includes custom fields.
  3. Allows addition of notes.
  4. Conditionally sends automated e-mails.
For a vendor:
  1. Forces screening to be current before any modifications can be saved.
  2. Allows addition of notes.
  3. Conditionally sends automated e-mails.
IntuitiveUI_ItemCard_CUSTOM The modified item card includes the custom tables:
IntuitiveUI_MasterControl_CUSTOM Master Controls Explorer.
IntuitiveUI_PurchaseOrder_CUSTOM The modified purchase order card:
  1. Forces screening to be current for the vendor before a new order can be created or any modifications can be saved on an existing order.
  2. Contains a quantity breakdown for a customer order and stock.
  3. Permits entry of setup and tooling charges on the item line.
  4. Allows addition of notes and phrases.
  5. Checks for foreign vendors.
  6. Checks for defense articles.
  7. Conditionally sends automated e-mails.

The modified purchase order receipt card:
  1. Forces the counted by field to be required.
  2. Create a text file compatible with the label printer for printing labels and send it to the printer.
IntuitiveUI_Quote_CUSTOM The modified quote card:
  1. Forces screening to be current for the customer before a new quote can be created or any modifications can be saved on an existing quote.
  2. Allows addition of phrases.
  3. Conditionally sends automated e-mails.
IntuitiveUI_SalesOrder_CUSTOM The modified sales order card:
  1. Forces screening to be current for the customer before a new quote can be created or any modifications can be saved on an existing order.
  2. Collects end user information when required.
  3. Includes custom fields.
  4. Allows the selection of a certificate.
  5. Automatically populates the item and price based on the selected quote.
  6. Allows addition of notes and phrases.
  7. Checks for customers with foreign addresses.
  8. Requires any added items to have a valid use class.
  9. Conditionally sends automated e-mails.
IntuitiveUI_Shipment_CUSTOM The modified shipment card:
  1. Forces screening to be current for the customer before a shipment can be entered.
  2. Forces screening to be current for the end user before a shipment can be entered.
  3. Creates certificates when a shipment is saved.
  4. Includes custom fields.
  5. Conditionally sends automated e-mails.
IntuitiveUI_ShopDataMaintenance_CUSTOM The modified shop data maintenance:
  1. Forces the user to enter a password before entering time and/or quantities.
Notes

These forms can not be loaded and shown. They will error out and not display. The framework must be used to display them with LaunchMenuItem.

All events should have basic error trapping. Below is an example of the minimum recommended. It uses Intuitive's form to display errors and also writes out the error to the database including the class and function where the error was trapped.

VB 
Try

    'Code goes here.

Catch ex As Exception
    HandleException(ex)
    RotairDataAccess_Wrapper.Database.WriteToErrorTable(ex.TargetSite.ReflectedType.Name, _
                ex.TargetSite.Name, ex.Message)

End Try
