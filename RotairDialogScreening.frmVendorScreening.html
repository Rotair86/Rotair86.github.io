Remarks

This form is used because when a form is launched from the menu the intuitive framework doesn't provide a way to pass a parameter to a windows form, doesn't return a usable reference nor does it make it clear how to show an imsForm as a dialog.

The screening form needs to be shown as a modal dialog mainly to avoid extra complications. If the user can launch another screening dialog then it would either have to check if another is running on launch and stop itself from showing. This can confuse the user if they can't find the original which can happen in an MDI application. Another option would be for the form to reload itself when it receives the focus. This can cause problems depending on how the user changes focus because sometimes the mdichild got focus event fires and sometimes the mdiparent event fires. If no checks are done then the user can add a record in one form and not see it in the other. Deletes cause similiar problems. Making it a modal dialog solves these problems.

Example

This is how to add the party screening form for vendors to a menu item called 'Denied Party Screen (Vendors)'.

SQL 
EXEC sp_IMS_CreateMenuItemHeader
	@ItemKey = 'PartyScreenVendor',
	@MenuItemGroup = 'SalesOrderProcessing',
	@MenuType = 'Form',
	@DataType = 'Custom',
	@Parameter = 'Vendor',
	@Caption = 'Denied Party Screen (Vendors)',
	@LanguageID = 1,
	@Assembly = 'RotairDialog_Screening',
	@Type = 'RotairDialog_Screening.frmVendorScreening',
	@Custom = 0,
	@AccessMDB = NULL,
	@AccessObject = NULL,
	@AccessFunction = NULL,
	@ProcessType = 1,
	@ExternalAppPath = NULL,
	@DotNetMethod = NULL
