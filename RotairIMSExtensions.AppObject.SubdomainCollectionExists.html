SubdomainCollectionExists Method

Returns TRUE if the passed in subdomain collection exists as a subdomain.

Syntax
VB 
'Declaration
Public Function SubdomainCollectionExists( _
        AO As imsAppObject, _
        SubdomainType As String _
) As Boolean

Parameters

AO
Type: imsAppObject
Not passed in by the caller. This identifies what object the function is applicable to.
SubdomainType
Type: System.String
The subdomain whose existance we wish to check.

Return Value

Type: System.Boolean
True if the collection named by SubdomainType exists.
Remarks

Provides an easy way to check for a subdomain. Needed because when the system ctreates an object such as a sales order for a RMA it does not create any custom subdomains.

Example

The example sets a field only if the subdomain exists.

VB 

Dim ExportData As imsAO_ExportData
Dim colExportData As imsAO_ExportData_Collection

If aoSalesOrder.SubdomainCollectionExists("IntuitiveAO_SalesOrder_CUSTOM.imsAO_ExportData_Collection") Then
    colExportData = DirectCast(Subdomains(imsAO_ExportData.DomainName), imsAO_ExportData_Collection)
    ExportData = DirectCast(colExportData(0), imsAO_ExportData)
    ExportData.SetValue(imsAO_ExportData.FieldNames.fExLicHold, True)
End If
