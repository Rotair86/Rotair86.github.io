InvalidHandle Property

Constant for a row without an underlying object.

Syntax
VB 
'Declarations
Public Property InvalidHandle As Integer

Remarks

Rowhandle for a new row without a corresponding object. This is returned when the first field of a row is being edited. Until the value is accepted there is no object assicaited with this row. Once the first value is accepted (by pressing enter, tab or any other method to accept the value) an object is instantiated and a valid row handle will be returned on subsequent actions.

Example

This example shows how this property is used. Since a note needs a parent id and a parent id isn't created until the underlying object is created a note can't be created until a grid row is created which will in turn create the underlying object (which will create its id). This means that a note can be added anytime EXCEPT as the first field. So when the Notes button is pressed we check the rowhandle. If the row has not been created, we get a constant that matches the InvalidHandle property and we display a message instead of creating the note.

VB 
Private Sub GridView_Notes_ButtonPressed( _
            ByVal sender As Object,
            ByVal e As DevExpress.XtraEditors.Controls.ButtonPressedEventArgs) _
            Handles mNotesButton.ButtonPressed

    If GridView.FocusedRowHandle <> UI.InvalidHandle Then
        'Process the Note
    Else
        MessageBox.Show("Can not add a note until the grid row is created")
    End If

End Sub
