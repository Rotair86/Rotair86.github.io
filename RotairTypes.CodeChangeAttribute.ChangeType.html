ChangeType Enumeration

Used for easily choosing a reason for a code change.

Members
  Member name Description
Unspecified Should never have this value, if we do then there is a coding error somewhere. This entry exists because by default, numeric variables are initialized to 0 before they ever get set. If we have a valid value for 0 it could make if difficult to detect a bug. So this ensures that a vaiable that does not get explicity set does not contain a valid value.
Bug_Squish Corrects a bug.
Database_Change Caused by some kind of change originating in the database.
Enhancement_Coder Programmer recommended change.
Enhancement_User User recommended change.
Hotfix_Required Caused by a hotfix.
New_Code New functionality or feature.
Upgrade Some component was upgraded.
