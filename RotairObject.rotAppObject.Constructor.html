Remarks

A rotAppObject is not directly creatable. It must be created by the derived class. Either during or right after creation the delegates need to be set. They allow communication from the base to the data access layer while being de-coupled from the base class.

Examples

Below is an example of a constructor for a derived class. Note that after the call to MyBase the delegates get set. The text "Domain" has been substituted for the actual domain name.

While setting the delegates is not part of the constructor and is not required for the class to be created it is required for communication to the data access layer. This makes it mandatory 99% of the time. The delegates weren't added to the constructor because the number of them would make the code more difficult to read and maintain.

VB 
'A derived class instantiating the base.
Public Sub New(ByVal bReadonly As Boolean, ByVal bExists As Boolean)

    MyBase.New(strDomain, False, bReadonly, bExists)
    DeleteLink = AddressOf DataAccess.Domain.Delete
    GetExceptionLink = AddressOf DataAccess.Scalar.GetException
    LoadLink = AddressOf DataAccess.Domain.Load
    SaveLink = AddressOf DataAccess.Domain.Save

End Sub
