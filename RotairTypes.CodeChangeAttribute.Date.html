Date Property

Date the change was started.

Syntax
VB 
'Declarations
Public Shared Property Date As String

Remarks

This property is not really usable by the programmer due to the nature of the attributes. It is passed as s string to make entry easier. This should be the date work has started since during the course of a mod many objects can be affected. It is much simpler to add a new date that has already been determined then keep changing the date or try to remember to go back and change all the dates when it goes into production.

Example

This example adds a function to charge a different sales tax depending on the region. Since this happens behind the scenes, we put a code change attribute on the dll so we can verify that the changes are in effect because we do not have a visual representation of it. The Date is the second argument.

VB 
<CodeChange("JPM", "03/25/2013", Rotair_Types.ChangeType.Enhancement_User, Comments:="Change the sales tax for a specific region.")> _
Public Class imsAO_RegionalInvoice

    Public Sub CalculateSalesTax()

       If mstrRegion <> "2" Then
           mdblTotal = mdblCost * mconTaxRate1
        Else
           mdblTotal = mdblCost * mconTaxRate2
       End If

    End Sub

End Class
