DBConstants Module

Constants local to the RotairDataAccess namespace.

Namespace: RotairDataAccess
Assembly: RotairDataAccess.dll

The DBConstants module isn't exposed and doesn't expose any members.

Constants

There are many more constants than are listed here; the following is just an example.

The constants are grouped by table or domain. A domain is further grouped by table. A domain may or may not coorespond to an intuitive domain.

Inside each group can be further groupings of parameters, change tracking fields, values and/or logical fields.

Example 1

The CageCode group represents a single table; tblCageCode. Comments separate the stored procedure from the tables and fields. There is a grouping inside this group to contain the parameters. When a table has parameters for all fields that usually means inserting of new rows is done via the RotairDataAccess namespace.

Example 3

The RFQs group represents a domain consisting of the tables tblRFQMaster and tblRFQVendor. Four other tables will be in this domain (tblRFQItem, tblRFQItemQty, tblRFQVendorCharge and tblRFQVendorQuote) when the code updating these tables is moved over to .NET. Since these tables are still updated in Access only a subset of fields are needed. The RFQs group is presented to show how a domain is represented and also to show the value constants. These represent data that needs to be hard-coded.
