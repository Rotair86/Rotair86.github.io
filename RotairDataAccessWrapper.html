Remarks

The only purpose of this class is to allow data access to the UI where there aren't any application objects. The reasons there aren't any application objects are that at least one doesn't logically exist (DAtabase), others weren't created and may not be (creating an app object for a small amount of functions and no rules may not be nmore beneficial than a wrapper) and lastly some functions are not clearly suited for one domain as opposed to another.
