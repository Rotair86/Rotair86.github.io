Remarks

This is not a programmer usable class. It can be thought of as a system class but that is not entirely correct. Basically it is created for a class and serves to store who made a change and when, possibly adding comments relating to code changes in the complied dll. It is recommended to only store significant changes (such as architectual changes) or changes that may not be easily apparent.

Note that the comments get passed as a named parameter. This is done to make them optional incase a dll is needed to be marked with just a author, date and source. It is not, however, recommended to add an attribute without a comment.

Example

This example adds a function to charge a different sales tax depending on the region. Since this happens behind the scenes, we put a code change attribute on the dll so we can verify that the changes are in effect because we do not have a visual representation of it.

VB 
<CodeChange("JPM", "03/25/2013", Rotair_Types.ChangeType.Enhancement_User, Comments:="Change the sales tax for a specific region.")> _
Public Class imsAO_RegionalInvoice

    Public Sub CalculateSalesTax()

       If mstrRegion <> "2" Then
           mdblTotal = mdblCost * mconTaxRate1
        Else
           mdblTotal = mdblCost * mconTaxRate2
       End If

    End Sub

End Class
