Custom Base Classes

.Net redefines the "Base" class. Traditionally base class refers to the first class in the hierarchy. If we use this definition in .NET then we can never create a base class since every class inherits from System.Object. Also since multiple inheritance per class is unsupported we have to make concessions to acheive the desired results.

For example, using the traditional system we would have an AppObject base class and a LockingObject base class. Then a class can inherit AppObject and optionally inherit LockingObject if that is required. Since this is not permitted in .NET, what I have done is create an AppObject class marked MustInherit which I will refer to as a base class. I also created AppLockingObject which inherits AppObject and is also marked MustInherit so I will consider it a base class for this documentation even though technically neither one of them is a base class.
