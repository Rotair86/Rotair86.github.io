Remarks
A rotAppObjectCollection is a collection of rotAppObjects.
Example
To use just inherit like the example in the syntax section above. Then in the client you can add objects to the collection. In the examples the derived class is rotCustomer and the collection is rotCustomerCollection. The first example shows the simplest way to add an object (customer) to the collection.

Example 1

VB 
'Creating a new customer and adding it to the collection.
Private Sub ProcessNewRecord()
    Private NewCustomer As New rotCustomer
    Private CustomerCollection As New rotCustomerCollection

    NewCustomer.Name = "Joe"
    CustomerCollection.Add(NewCustomer)
    'Do some processing

End Sub

The second example demonstates polymorphic behavior. The second function can be in the same assembly or it can be in a different assembly that doesn't have a reference to rotCustomer or rotCustomerCollection. Instead it has references to rotAppObject and rotAppObjectCollection. Yet it can still create a new rotCustomer, alter a property and add it to the collection. Note that the properties need to be referenced by a sring. You typically wouldn't set properties this way, a property like customer name is too specific for such a general fucntion. It is only there to demonstrate how to set properties when you don't have access to the interface.

Example 2

VB 
'Creating a new customer and adding it to the collection without references.
Private Sub ProcessNewRecord()
    Private NewCustomer As rotCustomer
    Private CustomerCollection As New rotCustomerCollection

    NewCustomer = AddNewRecord(CustomerCollection)
    'Do some processing

End Sub


Private Function AddNewRecord(AnyCollection As rotAppObjectCollection) As rotAppObject
    Private NewRecord As rotAppObject

    NewRecord = AnyCollection.MemberFactory
    NewRecord.Field("Name") = "Joe"
    Return  NewRecord

End Function
