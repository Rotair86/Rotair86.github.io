MainAO.Bound Event

Binds the subdomains of the item to the binders.

Syntax
VB 
'Declaration
Protected Overrides Sub MainAO_Bound
Remarks

Sets the AppObject property of the binders for the subdomains ItemSupplemental and ItemUseClass. Also does a lookup to display a name instead of an id.

Example

Binds the ItemUseClass subdomain and looks up an employee name by the id in the table tblItemUseClass.

VB 
Protected Overrides Sub MainAO_Bound( _ 
    ByVal semder As System.Object, _
    ByVal e As System.EventArgs _
    )

    Dim strName As String = String.Empty
    Dim UseClass As imsAO_UseClass
    Dim colUseClass As imsAO_UseClass_Collection


    'Binds the subdomain.
    Me.bdrItemSupplemental.AO = DirectCast(MainAO.AO, imsAO_Item_CUSTOM).ItemSupplemental_Collection(0)
    If Not Me.bdrItemSupplemental.AO Is Nothing Then Me.bdrItemSupplemental.Bind()

    'These functions and events sometimes fire when the objects are nothing so we need to check.
    If Me.bdrUseClass.AO IsNot Nothing Then
        'Subdomains are stored in collections even if there is a 1 to 1 relationship.
        colUseClass = CType(MainAO.AO.Subdomains("ItemUseClass"), imsAO_UseClass_Collection)
        'We can jump right to the first element since there can only be one item in the collection.
        UseClass = CType(colUseClass(0), imsAO_UseClass)
        'This next line is best read from right to left
        'Retrieves the VerifiedBy field, passes that result (which is an id) to a function that will return the name.
        strName = DataAccessWrapper.Employee.NameByID(Enumerations.eNameReturnStyle.FirstLast, UseClass.GetString("VerifiedBy"))))
        'Display the name.
        txtVerifiedBy.Text = strName
    End If

End Sub
