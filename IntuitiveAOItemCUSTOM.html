Remarks

The only reason hotfixes are broken is because the copy item and create item revision tasks are not extensible. In order to get them to work requires changes to the code in such a way as to become over written if a hotfix touches specific classes. I do not even know if we use the copy item function. I do not believe that we are using the create item revision function. So it is conceivable that we can make these classes fully patchable if we wish to not support the copy item and create item revision functions. No one is complaining either way so I am going to leave it as is.
